# Git提交规范

**生效时间**: 2026-02-20 16:47 GMT+8
**要求人**: 主人

## 规则

**每次功能修改，必须提交到GitHub！**

## 工作流程

1. 修改代码
2. 本地测试
3. `git add` 修改的文件
4. `git commit` -m "描述"
5. `git push clean master` （推送到干净仓库）
6. `gh release create vx.x.x` （创建Release版本）

## 目的

- 方便回滚
- 版本追踪
- 代码备份
